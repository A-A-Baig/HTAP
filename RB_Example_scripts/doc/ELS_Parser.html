    class ELS_Parser

    A text-processing parser that does ASCII-only Equidistant Letter Sequence analyses similar to that described at en.wikipedia.org/wiki/Equidistant_letter_sequencing

    For my example, I use Moby Dick taken from Project Gutenberg, gutenberg.org.

    Constants

    DEFAULT_SEARCH_PARAMS

    I used values found at Brendan McKay's page at cs.anu.edu.au/~bdm/dilugim/moby.html as starting points to test my own program.

    Note some subtle differences - my values are 0-based (where a skip of 0 means “go to the next letter”), whereas Prof. McKay defines that as a skip of 1. There is a similar difference with regard to starting points. He also accomplishes searches for backward terms using a negative skip value, while I do a “positive skip” search for a reversed term.

def reset_params(search_params)
  @search_params            = search_params
  @search_params[:end_pt] ||= (@contents.size-1)
  # ||= for :end_pt allows nil for 'end of file'
  return self # return self so we can chain methods
end

def search(term=@search_params[:term])
  @search_params[:term] = term
  reversed_term = term.reverse
  warn "Starting search within #{@filename} " + 
    "using #{@search_params.inspect}" if ($DEBUG)
  final_start_pt = @search_params[:end_pt] - @search_params[:term].size
  @search_params[:start_pt].upto(final_start_pt) do |index|
    @search_params[:min_skip].upto(@search_params[:max_skip]) do |skip|
      candidate = construct_candidate(index, skip)
      
      if (candidate == @search_params[:term])
        return report_match(skip, index)
      end
      
      if (candidate == reversed_term)
        return report_match(skip, index, 'reversed ')
      end

    end
  end
  return report_match(false, false)
end
